# GEO/AEO 工具评测：geo-aeo-tracker 开源工具深度解析

在企业纷纷布局 GEO/AEO 的今天，如何监控品牌在 AI 搜索引擎中的表现？今天给大家评测一款开源免费、功能强大的 GEO/AEO 追踪工具——**geo-aeo-tracker**。

<Callout type="info">
本文基于 GEO-AEO-Library 知识库的实战资料整理，工具评测来自真实使用体验。
</Callout>

## 为什么需要 GEO/AEO 追踪工具？

传统 SEO 时代，我们用 Google Analytics、Ahrefs、SEMrush 监控关键词排名和流量。但在 AI 搜索时代，这些工具已经不够用了：

<Callout type="warning">
**传统 SEO 工具的局限性**：
- ❌ 无法追踪品牌在 ChatGPT、Perplexity、Gemini 等 AI 引擎中的表现
- ❌ 无法分析 AI 引擎如何引用和提及品牌
- ❌ 无法对比竞争对手的 AI 可见性分数
- ❌ 无法发现 AI 搜索中的引用机会
</Callout>

这就是 geo-aeo-tracker 诞生的原因。

## geo-aeo-tracker 核心功能

这款工具提供 **12 个功能标签页**，覆盖 GEO/AEO 工作的完整流程：

### 1. 项目设置 ⚙️
配置品牌名称、关键词、描述，作为后续所有追踪的基础。

### 2. 提示词中心 💬
管理和组织追踪提示词，支持批量导入和分类管理。

### 3. 角色扩散 🎭
生成角色特定的提示词，模拟不同用户画像的查询行为。

### 4. 细分领域探索器 🔍
自动生成高意图查询，帮助发现长尾关键词和场景化问题。

### 5. 响应浏览 📝
浏览各个 AI 引擎的实时响应，了解品牌如何被提及和描述。

### 6. 可见性分析 📊
<Badge color="green">核心功能</Badge> 趋势图表展示品牌可见性分数变化，支持多维度对比。

**可见性评分算法（0-100分）基于**：
- 品牌提及（Mention）
- 位置（Position）
- 频率（Frequency）
- 引用（Citation）
- 情感（Sentiment）

### 7. 引用分析 🔗
分析 AI 引擎引用品牌内容的频率和方式。

### 8. 引用机会 🎯
<Badge color="blue">实战价值</Badge> 发现竞争对手被引用但你的品牌未被引用的机会点。

### 9. 竞争对手对战卡 ⚔️
AI 自动生成的品牌对比分析，了解在 AI 眼中你与竞品的差异。

### 10. AEO 审计 🏥
站点 AEO/GEO 准备度检查，评估内容结构化程度。

### 11. 自动化 ⏱️
支持 Cron 和 GitHub Actions，实现定期自动追踪和报告。

### 12. 文档 📖
包含 14 部分的完整使用指南，新手友好。

## 技术架构与支持平台

<Callout type="tip">
**技术栈**：
- **框架**: Next.js 16.1 + Turbopack（极速开发体验）
- **语言**: TypeScript（类型安全）
- **样式**: Tailwind CSS v4（现代化 UI）
- **存储**: IndexedDB + localStorage（本地优先）
- **抓取**: Bright Data API（专业爬虫服务）
- **LLM**: OpenRouter（多模型支持）
</Callout>

**支持的 AI 模型**：
- ChatGPT
- Perplexity
- Gemini
- Copilot
- Google AI Overview
- Grok

## 快速开始指南

<Steps>
  <Step>
    <h4>克隆项目</h4>
    <p>从 GitHub 克隆 geo-aeo-tracker 项目到本地。</p>
    ```bash
    git clone https://github.com/danishashko/geo-aeo-tracker
    cd geo-aeo-tracker
    ```
  </Step>
  <Step>
    <h4>安装依赖</h4>
    <p>使用 npm 或 pnpm 安装项目依赖。</p>
    ```bash
    npm install
    ```
  </Step>
  <Step>
    <h4>配置环境</h4>
    <p>复制环境变量模板并配置 API 密钥。</p>
    ```bash
    cp .env.example .env.local
    # 编辑 .env.local，添加 Bright Data 和 OpenRouter API 密钥
    ```
  </Step>
  <Step>
    <h4>启动服务</h4>
    <p>启动本地开发服务器，访问 http://localhost:3000。</p>
    ```bash
    npm run dev
    ```
  </Step>
  <Step>
    <h4>部署上线</h4>
    <p>使用 Vercel 一键部署到生产环境。</p>
    ```bash
    vercel --prod
    ```
  </Step>
</Steps>

## 实战使用建议

### 初次使用

1. **创建项目**：设置品牌信息和核心关键词
2. **添加提示词**：导入 20-50 个高意图查询
3. **运行批量抓取**：获取初始数据基线
4. **分析可见性分数**：了解当前表现

### 定期监控

1. **设置 Cron 任务**：每周自动抓取一次
2. **监控分数趋势**：关注上升或下降趋势
3. **发现引用机会**：优化内容填补空白
4. **调整内容策略**：基于数据优化方向

<Callout type="warning">
**API 费用预估**：
- Bright Data: $100-300/月（取决于抓取量）
- OpenRouter: $10-50/月（取决于模型调用次数）
- 总计: 约 $110-350/月
</Callout>
