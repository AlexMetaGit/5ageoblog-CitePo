# GEO/AEO 完全指南：从 SEO 到 AI 时代的品牌可见性革命

<Callout type="info">
<strong>导读</strong>：AI 搜索正在重塑品牌可见性的规则。本文基于 GitHub Top 10 GEO/AEO 开源项目的完整调研，带你从零开始掌握 AI 搜索优化的核心技术和实践策略。
</Callout>

在 ChatGPT 占据 AI 推荐流量 87.4% 的时代，品牌正面临前所未有的可见性挑战。传统 SEO 专注于排名和点击，而 GEO/AEO 则聚焦于**信任**和**引用**——确保 AI 在生成答案时选择并引用你的内容。

## 什么是 GEO/AEO？

### 核心定义

**GEO（Generative Engine Optimization）**：为生成式搜索引擎优化的策略体系

**AEO（Answer Engine Optimization）**：为答案引擎优化的实践方法

### 与 SEO 的根本区别

| 维度 | 传统 SEO | GEO/AEO |
|------|---------|---------|
| **目标** | 提高搜索排名 | 成为 AI 引用的可信来源 |
| **机制** | 关键词匹配 | 内容信任度评估 |
| **结果** | 点击率（CTR） | 引用频率和可见性 |
| **用户行为** | 4.2 词平均搜索 | 23 词复杂提示词 |

<Callout type="warning">
关键认知：AI 不会列出 10 个结果，而是选择少数可信来源进行总结。被引用 = 被发现。
</Callout>

## GEO 的三大支柱

### 1. llms.txt - AI 内容索引标准

基于 GitHub 排名第 1 的开源项目（2,216 ⭐），为 AI 提供标准化的内容索引。

**文件结构示例**：
```
---
title: Your Website Name
version: 1.0
updated: 2026-03-30

# Content Sections
## Documentation
- Getting Started: /docs/getting-started
- API Reference: /docs/api
- Tutorials: /docs/tutorials

## Data
- Dataset: /dataset
- Model Weights: /models
---
```

**实施步骤**：
1. 在网站根目录创建 `/llms.txt`
2. 添加元数据（标题、版本、更新时间）
3. 组织内容索引（文档、API、数据集）
4. 定期更新内容

### 2. Schema.org 结构化数据

为 AI 提供结构化的内容理解：

```json
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "文章标题",
  "author": "作者",
  "datePublished": "2026-03-30",
  "articleSection": "内容分类"
}
```

## 关键成功指标

### 可见性评分（0-100）

基于以下维度的综合评分：
- **品牌提及**（Mention）：品牌在 AI 回答中被提及的次数
- **位置**（Position）：在 AI 回答中的出现位置
- **频率**（Frequency）：被引用的频率
- **引用**（Citation）：被直接引用的次数
- **情感**（Sentiment）：提及的情感倾向

### 行动目标

<Callout type="tip">
根据 GitHub 调研数据，以下目标值得追求：
- 内容更新时间 < 3 个月的被引用概率高 3 倍
- AI 推荐转化率是传统来源的 2 倍
- ChatGPT 中约 24% 的响应无需获取在线内容
</Callout>

## 常见误区与解决方案

### 误区 1：GEO 就是新的 SEO

** reality：** GEO 关注的是信任和引用，而非关键词和排名。

**解决方案：** 建立内容权威性，而非关键词密度优化。

### 误区 2：AI 会完全取代传统搜索

** reality：** AI 搜索与传统搜索将长期共存，互补发展。

**解决方案：** 同时优化 SEO 和 GEO，覆盖不同用户习惯。

### 误区 3：只要做好技术就能成功

** reality：** GEO 需要内容、技术和数据的综合优化。

**解决方案：** 建立完整的工作流程，包括内容创作、技术实施和数据监控。
