# GEO 工具实战：如何追踪品牌在 AI 搜索中的可见性

<Callout type="info">
  <strong>核心问题</strong>：你的品牌在 ChatGPT、Perplexity 等 AI 搜索中被推荐了吗？如果没有，你甚至不知道自己"不可见"。
</Callout>

## 为什么你需要 AI 可见性追踪？

<Badge color="red">震撼数据</Badge> 根据 GEO Guide 研究报告：

- **88%** 的品牌在 AI 搜索中是不可见的（即使传统搜索排名靠前）
- ChatGPT 占据 AI 推荐流量的 **87.4%**
- AI 推荐的转化率是传统搜索的 **2 倍**

**更关键的是**：传统 SEO 工具无法追踪 AI 可见性。你需要专门的 GEO 工具。

## 快速开始：5 步部署你的追踪系统

### 前置要求

- Node.js 18+ 环境
- Bright Data API 密钥（用于抓取 AI 响应）
- OpenRouter API 密钥（用于 LLM 分析）

### 步骤 1：克隆项目

```bash
git clone https://github.com/danishashko/geo-aeo-tracker
cd geo-aeo-tracker
```

### 步骤 2：安装依赖

```bash
npm install
```

### 步骤 3：配置环境变量

```bash
cp .env.example .env.local
```

编辑 `.env.local`：

```env
# Bright Data 配置
BRIGHT_DATA_API_KEY=your_key_here

# OpenRouter 配置
OPENROUTER_API_KEY=your_key_here

# 可选：自定义端点
NEXT_PUBLIC_APP_URL=http://localhost:3000
```

<Callout type="warning">
  <strong>成本提示</strong>：Bright Data 提供免费试用额度，OpenRouter 按 token 计费（GPT-4 约为 $0.03/1K tokens）。月度预算约 $10-50，取决于追踪频率。
</Callout>

### 步骤 4：启动服务

```bash
npm run dev
```

访问 `http://localhost:3000`，你将看到仪表板界面。

### 步骤 5：创建第一个项目

1. 点击"New Project"
2. 填写品牌信息：
   - 品牌名称：例如 "Notion"
   - 关键词：`["notion", "productivity tool", "note-taking"]`
   - 描述：简短的品牌介绍
3. 添加追踪提示词（建议从 10-20 个开始）：
   - "最好的项目管理工具"
   - "Notion vs Asana 对比"
   - "远程团队协作工具推荐"
4. 运行首次抓取
5. 等待分析完成（约 5-10 分钟）

## 实战应用场景

### 场景 1：品牌监控

**目标**：实时追踪品牌在 AI 平台中的表现

**操作**：
1. 设置品牌相关提示词（20-50 个）
2. 配置每日自动抓取（Cron: `0 9 * * *`）
3. 监控可见性分数变化
4. 发现分数下降时及时调查

**关键指标**：
- 平均可见性分数
- 引用频率趋势
- 情感分析（正面/中性/负面）

### 场景 2：竞争分析

**目标**：了解竞争对手的 AI 搜索表现

**操作**：
1. 在"引用机会"模块查看对比数据
2. 使用"竞争对手对战卡"生成差异化内容
3. 针对对手被引用但你没有的提示词优化内容

**案例**：
```
发现：在"AI 写作工具推荐"提示词中
- Jasper 被引用 8 次/10 次查询
- Copy.ai 被引用 6 次/10 次查询
- 你的品牌 被引用 0 次/10 次查询

行动：创建"AI 写作工具对比"内容，突出你的优势
```

### 场景 3：内容优化

**目标**：基于 AI 可见性数据优化内容策略

**操作**：
1. 使用"细分领域探索器"发现长尾机会
2. 在"响应浏览器"中分析 AI 回答结构
3. 创建符合 AI 偏好的内容格式：
   - 清晰的问答结构
   - 实体标记（产品名、功能、价格）
   - 权威来源引用

**最佳实践**：
- AI 偏好**结构化**内容（列表、表格、步骤）
- AI 偏好**权威来源**（研究报告、官方文档）
- AI 偏好**更新频繁**的内容（少于 3 个月）

## 高级技巧：提升追踪效率

### 技巧 1：提示词分层策略

不要只用品牌词，建立三层提示词体系：

```
Layer 1 - 品牌词（10%）
"什么是 [品牌名]"
"[品牌名] 怎么样"

Layer 2 - 产品词（30%）
"最好的 [产品类别]"
"[产品类别] 推荐"

Layer 3 - 场景词（60%）
"如何解决 [痛点]"
"[场景] 工具推荐"
```

**原因**：80% 的 AI 搜索是场景驱动的，而非品牌驱动。

### 技巧 2：角色扩散批量生成

使用 AI 批量生成角色特定提示词：

```bash
# 示例：生成 10 个角色的提示词
基础词："推荐最好的 CRM 工具"

角色列表：
- 销售经理
- 创业公司创始人
- 电商运营
- B2B 市场
- ...
```

**效果**：覆盖更多用户场景，发现隐藏机会。

### 技巧 3：自动化报告

配置 GitHub Actions 每周自动生成报告：

```yaml
# .github/workflows/geo-report.yml
name: Weekly GEO Report
on:
  schedule:
    - cron: '0 9 * * 1' # 每周一 9:00
jobs:
  report:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Run GEO Tracker
        run: npm run track
      - name: Send Report
        run: npm run send-report
```

**报告内容**：
- 可见性分数变化（周环比）
- 新增引用机会
- 竞争对手动态
- 优化建议
