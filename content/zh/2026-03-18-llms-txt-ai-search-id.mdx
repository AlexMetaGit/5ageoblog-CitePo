# llms.txt - 你的网站在 AI 搜索时代的"身份证"

## AI 搜索正在改写游戏规则

<Callout type="info">
  <Badge color="blue">关键数据</Badge> ChatGPT 已占据 AI 推荐流量的 **87.4%**，AI 推荐的转化率是传统来源的 **2 倍**。
</Callout>

如果你还在用传统 SEO 的思维优化网站，可能已经错过了新流量红利。用户行为正在发生根本性变化：

- 用户向 ChatGPT 输入 **23 个单词**的提示词
- 传统搜索平均只有 **4.2 个单词**
- 用户寻求**综合答案**而非链接列表

这意味着：**被 AI 引用 = 被发现**。

## 传统 SEO 不够用了

传统 SEO 关注：
- 关键词排名
- 反向链接
- 页面权重

但 AI 搜索引擎需要的是：
- **结构化的内容索引**
- **清晰的语义标记**
- **可信的来源引用**

<Callout type="warning">
  权威现在在 AI 模型内部，不仅仅在 Web 上。AI 模型的训练数据决定了你的内容是否会被引用。
</Callout>

## llms.txt - AI 时代的 robots.txt

就像 `robots.txt` 告诉搜索引擎如何抓取你的网站，`llms.txt` 告诉 AI 模型如何理解你的内容。

### 什么是 llms.txt？

`llms.txt` 是一个标准化的文件格式，位于网站根目录（`/llms.txt`），为大语言模型提供内容索引。由 Answer.ai 提出，目前在 GitHub 上已有 **2,216 ⭐**。

### 核心功能

<Badge color="green">标准化格式</Badge> `/llms.txt` 和 `/llms-full.txt` 两种文件

<Badge color="blue">LLM 友好</Badge> 为 ChatGPT、Claude、Gemini 提供索引

<Badge color="purple">灵活扩展</Badge> 支持文档、API、数据集等多种内容类型

## 如何实施 llms.txt

<Steps>
  <Step title="创建文件">
    在网站根目录创建 `/llms.txt` 文件：
    
    ```bash
    # 在网站根目录
    touch llms.txt
    ```
  </Step>

  <Step title="添加元数据">
    ```yaml
    ---
    title: Your Website Name
    version: 1.0
    updated: 2026-03-18
    ---
    ```
  </Step>

  <Step title="组织内容索引">
    ```markdown
    # Content Sections
    
    ## Documentation
    - Getting Started: /docs/getting-started
    - API Reference: /docs/api
    - Examples: /docs/examples
    
    ## Resources
    - Blog: /blog
    - Tutorials: /tutorials
    - Case Studies: /cases
    
    ## Data
    - Dataset: /dataset
    - API Endpoint: /api/v1
    ```
  </Step>

  <Step title="定期更新">
    内容更新时间少于 **3 个月**的网站，被 AI 引用的概率高 **3 倍**。建议：
    - 每月检查并更新 llms.txt
    - 添加新内容时同步更新索引
    - 使用自动化工具生成和维护
  </Step>
</Steps>
