# llms.txt - 新一代AI搜索标准规范

<Callout type="info">
  <strong>导读</strong>：随着ChatGPT、Claude等AI搜索引擎的崛起，传统的robots.txt已无法满足AI时代的内容索引需求。llms.txt作为专门为AI设计的网站标准，正在重新定义品牌在AI搜索中的可见性规则。
</Callout>

## 🚀 搜索范式的革命性转变

### 从链接时代到回答时代

互联网正在经历根本性变革：
- **传统SEO**：关注关键词排名和点击率
- **AI搜索GEO**：关注内容引用和信任建立

**关键数据**：
- ChatGPT占AI推荐流量的87.4%
- AI推荐转化率是传统来源的2倍
- 88%的品牌在AI搜索中不可见，即使它们在传统搜索中排名靠前

<Callout type="warning">
  传统SEO策略正在失效，HubSpot曾经历80%的流量下降，迫使其全面转向AEO策略。
</Callout>

### llms.txt的诞生背景

llms.txt由Answer.ai团队提出，类似于`robots.txt`，但专门为LLM（大语言模型）设计。该项目在GitHub上获得2,216⭐，是目前GEO/AEO生态系统中排名第一的项目。

## 📋 llms.txt核心功能解析

### 标准化格式设计

llms.txt采用简洁的YAML+Markdown混合格式，既保持机器可读性，又保证人类可理解性：

```
---
title: My Website
version: 1.0
updated: 2025-10-01

# Content Sections
## Docs
- Documentation: /docs
- API Reference: /api
- Tutorials: /tutorials

## Data
- Dataset: /dataset
- Model Weights: /models
---
```

### LLM友好的内容组织

<AccordionGroup>
  <Accordion title="为什么需要标准化格式？">
    标准化格式让AI引擎能够：
    - 快速理解网站结构和内容类型
    - 准确索引最重要的文档和数据
    - 避免抓取无关或低质量内容
  </Accordion>
  <Accordion title="与robots.txt的区别是什么？">
    robots.txt主要针对网页爬虫，阻止抓取；llms.txt则主动引导AI如何使用内容。
  </Accordion>
</AccordionGroup>

### 两种文件类型的区别

| 文件类型 | 用途 | 特点 |
|---------|------|------|
| `/llms.txt` | 标准索引 | 精简版，快速概览 |
| `/llms-full.txt` | 详细索引 | 完整版，深度内容 |

### 优化策略

#### Query Fan-Out技术

AI系统会将一个原始查询分解为8-12个子查询：

- **规格化**：寻找精确数据
- **概括化**：理解父类别
- **澄清化**：检查用户意图
- **后续化**：预测用户下一步

<Badge color="yellow">161%</Badge> - 针对子查询优化的页面获得AI引用的概率提升

#### E-E-A-T原则

建立AI信任的三个支柱：
- **专业性**：展示领域 expertise
- **权威性**：可验证的信息来源
- **可信度**：准确、及时的内容
